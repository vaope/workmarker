# WorkEventAgent 客户端 UX 设计 v1

## 产品原则

客户端是“捕获优先”的工具，不是“浏览优先”的项目管理软件。

最短成功路径应该是：

```text
唤起 -> 输入进展 -> 确认归档
```

项目浏览、项目校验、时间线回看都是辅助流程，不能让日常捕获变慢。

## 整体结构

```text
WorkEventAgent 客户端
├─ 主窗口
│  ├─ 项目库
│  ├─ 工作视图
│  │  ├─ 任务视图
│  │  └─ 时间线视图
│  └─ 常驻输入条
└─ 快速捕获窗口
   ├─ 文本输入
   ├─ 剪贴板图片附件
   └─ 内联确认卡片
```

## 主窗口

主窗口采用双栏结构：左侧是项目库，右侧是选中项目的任务和进展，底部是常驻输入条。

```text
+----------------------------------------------------------+
| WorkEventAgent                                           |
+------------+---------------------------------------------+
| 项目库      | [任务视图] [时间线]                  搜索   |
|            +---------------------------------------------+
| ● 多模态    | ▾ 使用 KV cache 优化 few-shot              |
|   标注系统  |   ● 查看当前阻塞点        [进行中]         |
|            |     -> 梳理推理链路...       3小时前        |
| ○ WorkE..  |   ● KV cache 原理解读     [进行中]         |
|            |     -> 整理 vLLM 源码...    昨天           |
| ○ 简历项目  |                                             |
|            | ▾ XX 场景下数据收集                         |
|------------|   ● 数据集生产            [进行中]         |
| [+ 新建]   |     -> 完成第二批标注...     2天前          |
|            |   ✓ 轨迹投影脚本编写      [已完成]         |
+------------+---------------------------------------------+
| [附件] 记录一下今天的进展...                  [提交 Enter] |
+----------------------------------------------------------+
```

## 项目库

左栏是固定宽度的项目库，建议宽度约 240px。

每个项目显示：

- 项目名称
- 未完成任务数
- 最近更新时间
- 选中状态：左侧色条和背景高亮

底部提供 `+ 新建`，用于打开项目初始化表单。

项目右键菜单：

- 打开项目目录
- 校验项目
- 从项目库移除，但不删除文件

## 工作视图

右侧工作视图支持两种模式：任务视图和时间线视图。

### 任务视图

任务视图是默认模式。

展示方式：

- 按事项分组
- 每个任务一行
- 每行展示任务名、状态、下一步摘要、最近更新时间

点击任务行后，在原地展开该任务的完整时间线。展开方式采用手风琴模式，一次只展开一个任务。

### 时间线视图

时间线视图按时间倒序展示归档事件。

每条事件展示：

- 时间
- 项目 / 事项 / 任务路径
- summary 摘要
- 是否有附件

这个视图后续可以服务日报、周报和最近进展回顾。

## 常驻输入条

底部输入条始终可见。

默认态：

- 单行输入
- 高度约 48px
- 附件按钮
- 提交按钮

聚焦态：

- 展开为多行输入，最高约 160px
- 支持粘贴图片
- 图片缩略图显示在输入框上方

键盘行为：

- `Enter`：提交
- `Ctrl+Enter`：换行
- `Esc`：收起或取消当前输入

## 快速捕获窗口

快速捕获窗口独立于主窗口，用于全局热键快速唤起。

```text
+-------------------------------------------+
| 快速捕获                         Esc 关闭 |
+-------------------------------------------+
|                                           |
| 今天看了 KV cache 的阻塞点，主要问题是    |
| prefix 复用策略不清楚，下一步梳理推理链路。|
|                                           |
| [图片] screenshot-0630.png  x             |
|                                           |
+-------------------------------------------+
|              [提交 Enter] [取消 Esc]       |
+-------------------------------------------+
| 上次：5分钟前 -> 多模态标注/KV cache/阻塞  |
+-------------------------------------------+
```

行为：

- 全局热键唤起窗口
- 默认热键建议：`Ctrl+Shift+Space`
- 唤起后立即聚焦输入区
- 窗口位于屏幕中央偏上
- 宽度约 480px
- 提交后，输入区被确认卡片替换
- 确认或取消后，窗口隐藏回托盘

底部一行显示最近一次归档结果，用于建立“系统确实工作了”的反馈。

## 确认卡片

确认卡片内联展示，不新开弹窗。

```text
+-------------------------------------------+
| 归档预览                       置信度 92% |
+-------------------------------------------+
| 项目  基于大模型的多模态标注系统          |
| 事项  使用 KV cache 优化 few-shot         |
| 任务  查看当前阻塞点                      |
|-------------------------------------------|
| 状态    进行中                            |
| 摘要    梳理推理链路，明确 prefix 复用策略 |
| 下一步  映射推理链，确认 prefix 边界       |
| 附件    screenshot-0630.png               |
+-------------------------------------------+
|        [确认]      [编辑]      [取消]      |
+-------------------------------------------+
```

三个操作：

- 确认：写入 Markdown 和 SQLite；如果来自快速捕获窗口，则短暂显示成功状态后隐藏窗口。
- 编辑：项目、事项、任务字段变成可编辑状态；编辑后回到确认态。
- 取消：回到输入态，并保留原始输入文本。

低置信度处理：

- 当置信度低于 70%，项目字段默认变成下拉选择。
- 显示黄色提示：`不太确定属于哪个项目，请确认`。
- 不允许低置信度结果绕过确认直接写入。

## 图片粘贴流程

图片支持通过剪贴板粘贴，不要求用户手动选择附件。

```text
用户 Ctrl+V
  -> 检测剪贴板类型
    -> 纯文本：正常粘贴
    -> 图片：
      1. 生成临时缩略图
      2. 在输入框上方显示图片标签
      3. 图片暂存到 %TEMP%/workeventagent/pending/
      4. 支持多张图片横向排列
      5. 支持删除单张图片

提交
  -> 只把文字交给 opencode
  -> 图片只作为附件路径保留
  -> 确认卡片展示附件列表

确认
  -> 图片移动到项目附件目录
  -> 推荐路径：attachments/{task_id}/{timestamp}-{index}.png
  -> 路径写入 Markdown
  -> 重建 SQLite
  -> 清理临时文件

取消
  -> 清理临时文件
  -> 回到输入态时保留原始文字
```

MVP 不把图片内容交给 LLM 解析。

## 项目初始化表单

项目初始化是用户显式创建项目，不等同于 agent 根据一句进展自动创建项目。

```text
+-------------------------------------------+
| 初始化新项目                              |
+-------------------------------------------+
| 项目名称  [基于大模型的多模态标注系统  ]  |
| 项目 ID   [multimodal-labeling         ]  |
|           自动生成，可手动修改             |
| 保存位置  [D:\worklogs\              选择]|
|                                           |
| 初始事项（可选）                           |
| [+ 添加事项]                              |
|   事项名  [使用 KV cache 优化 few-shot ]  |
|     [+ 添加任务]                          |
|       任务名 [查看当前阻塞点            ]  |
|       任务名 [KV cache 原理解读         ]  |
+-------------------------------------------+
|              [创建项目] [取消]             |
+-------------------------------------------+
```

客户端负责：

- 生成标准 Markdown
- 生成稳定 ID
- 创建附件目录
- 初始化或重建 SQLite
- 注册到项目库
- 关闭弹窗前校验生成文件

初始事项和任务不强制填写，但界面要强烈引导用户至少填写一个事项和一个任务。

## 状态反馈

| 场景 | 反馈 |
|---|---|
| 归档成功 | toast 显示约 2 秒：`已归档到 KV cache / 查看当前阻塞点` |
| opencode 调用中 | loading 状态：`正在归档...` |
| opencode 超时或失败 | 红色 toast，提供重试按钮 |
| 项目校验失败 | 弹窗列出具体问题和修复建议 |
| 全局热键冲突 | 设置页提示，并允许自定义热键 |

## 技术方向

推荐技术栈：

- Tauri v2 作为桌面客户端壳
- 当前 `workeventagent.exe` 作为 sidecar
- Markdown 和 SQLite 保持本地文件
- opencode 继续由用户在本机配置

选择原因：

- 可生成较小的桌面程序
- 支持窗口、托盘、热键、剪贴板、文件系统等原生能力
- 不重写已经验证过的 Python 归档核心
- UI 和归档逻辑保持分离

## MVP 范围

纳入 MVP：

- 项目库
- 项目初始化
- 任务视图
- 时间线视图
- 底部常驻输入条
- 快速捕获窗口
- 粘贴图片预览，并在确认后归档
- 确认卡片
- 低置信度修正流程

暂不纳入 MVP：

- 日报和周报生成
- 全文搜索
- 纠错事件 UI
- 多项目批量迁移
- 云同步
- 多设备支持
